Partie 1 – Chiffrement des mots de passe par Master Key Contexte • Dans TP3, les mots de passe étaient stockés en clair dans MySQL. • Cela n’est plus acceptable. • Désormais, une Master Key fournie par l’administrateur BP doit être utilisée pour chiffrer tous les mots de passe avant stockage. Exigences obligatoires • Le mot de passe ne doit jamais être stocké en clair. • Le mot de passe doit être chiffré avant insertion en base. • La Master Key ne doit jamais être dans le code. • La Master Key doit être injectée via variable d’environnement. Nom imposé : APP_MASTER_KEY • Si la clé est absente, l’application doit refuser de démarrer. Implémentation recommandée • Algorithme : AES en mode GCM. • AES GCM fournit chiffrement et intégrité. Format de stockage conseillé : • v1:Base64(iv):Base64( ciphertext ) • Structure MySQL mise à jour D. Samfat TP Serveur d'Authentification 25
